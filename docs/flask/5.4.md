5.4 部署：让全世界看到你的作品
你不能直接用 flask run 来部署正式网站，因为它性能弱且不安全。在生产环境中，我们需要一套“黄金组合”。
 * Gunicorn (WSGI 容器)： 负责运行多个 Flask 进程，处理并发请求。
 * Nginx (反向代理)： 负责接收外界请求，处理静态文件（CSS/JS），并将动态请求转发给 Gunicorn。
 * Docker (可选但推荐)： 把你的代码、环境、配置打包成一个容器，实现“一次构建，到处运行”。
部署命令示例 (Gunicorn)：
# 在服务器上运行，开启 4 个工作进程
gunicorn -w 4 -b 0.0.0.0:8000 app:app

📂 最终推荐的项目结构
一个成熟的 Flask 项目通常长这样：
/my_project
├── app/                  # 核心代码
│   ├── __init__.py       # 工厂函数
│   ├── models.py         # 数据库模型
│   ├── static/           # 静态文件
│   ├── templates/        # 模板
│   └── main/             # 蓝图目录
│       ├── __init__.py
│       └── views.py
├── tests/                # 测试用例
├── migrations/           # 数据库迁移记录
├── config.py             # 配置文件
├── requirements.txt      # 依赖包清单
└── run.py                # 启动入口
